介绍Java枚举，说明其使用的场景。

  • 目录

枚举类即该类的所有对象是有限的，比如说Week类，其实例只会有7个，周一、周二直到周日，那么就可以直接使用枚举类将其一次性实例化，如下所示：

// 定义一个星期的枚举类
public enum WeekEnum {
    // 在第一行显式地列出7个枚举实例(枚举值)，系统会自动添加 public static final 修饰
    SUNDAY, MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY; // 相当于一次性实例化了7个WeekEnum对象。
}

以上是使用默认构造器的枚举类，那么对于带构造函数的枚举类来说，则需要在定义对象的时候进行调用构造函数：

package enumtest;

public enum WeekEnum {
    
    // 需要传递参数使用自定义构造函数进行初始化对象。 实际是调用 private WeekEnum(String date) 构造方法。
    SUNDAY("星期日"), MONDAY("星期一"), TUESDAY("星期二"), WEDNESDAY("星期三"), 
    THURSDAY("星期四"), FRIDAY("星期五"), SATURDAY("星期六"); 

    // 定义一个 private 修饰的实例变量
    private String date;

    // 定义一个带参数的构造器，枚举类的构造器只能使用 private 修饰
    private WeekEnum(String date) {
        this.date = date;
    }

    // 定义 get set 方法
    public String getDate() {
        return date;
    }

    public void setDate(String date) {
        this.date = date;
    }
}

从上面可以看出，枚举类相当于是一种对只需要初始化有限个对象的类简化构造过程，不需要一个个进行new，而且其实例对象就在类中，对于上面的

SUNDAY("星期日"),

就类似于如下：

public final WeekEnum SUNDAY = new WeekEnum("星期日");

其实就是一种语法糖，能够更加方便的构造对象。使用枚举的前提是该类的实例是固定的，对于通用类来说，因为其实例需要的情况不能控制，所以不使用枚举。除了写法不同以外，其他的调用和普通的类没有区别。直接进行方法调用即可。另外，也可以使用枚举创建线程安全的单例对象，如下所示。

// 枚举实现单例模式。
public enum Singleton {
    INSTANCE;
    public void doSomething() {
        System.out.println("doSomething");
    }
}
